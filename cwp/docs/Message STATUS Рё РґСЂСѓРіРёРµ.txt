Сообщние STATUS CWP посылает FDP со списком выбранных логических??? секторов или
при выключении CWP с пустым списком.

Сообщние STATUS состоит из
 - тип сообщения (STS)
 - отправитель/получатель (не более 8 символов каждый, через слэш)
 - номер сообщения (000 - 999)
 - приоритет сообщения
 - ID станции (собственник секторов)
 - список секторов через тире (S1-S2-...-Sn), может быть пустым
 
На собщение STATUS отвечаем LAM ( или ERROR ).
  
Сообщние ERROR состоит из
 - тип сообщения (ERR)
 - отправитель/получатель (не более 8 символов каждый, через слэш)
 - номер сообщения (000 - 999)
 - ссылка на сообщение (отправитель/получатель и номер сообщения)
 - список кодов ошибок через тире (000 - 999) или доп. информация

Пример:
(STSCWPA/FDP012-1-CWPA-S1-S2)
(ERRFDP/CWPA013CWPA/FDP012-001-031)

(STSCWPA/FDP014-0-CWPA)
(LAMFDP/CWPA015CWPA/FDP014)


Сообщние INF состоит из
 - тип сообщения (INF)
 - отправитель/получатель (не более 8 символов каждый, через слэш)
 - номер сообщения (000 - 999)
 - тип сообшения INF
   - ACT
   - ASSUME
   - FPL
 - ACID
 - ADEP
 - ADES
 - CODE
 - сектор

Пример:
(INFCWPA/FDP012-ACT-AMM253-LMML-EGBB-1700-S1)


Сообщние FPL как ABI, но SSR Mode and Code = Fir name и Estimate Data = SPEED/LEVEL
